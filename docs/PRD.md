# 离线翻译 App PRD

## 产品目标

构建一个移动端优先的离线翻译 App。用户可以在无网络推理环境下完成短文本翻译，模型文件通过导入或下载进入 App 私有目录，运行时不依赖开发机路径。

## 当前版本范围

### v0.0.2

本版本确认并固化跨端需要复用的模型获取与模型选择体验：

- 用户只能通过“导入模型”或“下载模型”获得模型。
- 不支持用户手动输入模型路径。
- 模型导入或下载后，界面只显示模型名称，不显示完整路径。
- 模型名称展示区域不可编辑。
- 完整模型路径只允许保存在 App 内部状态或平台私有存储逻辑中。
- 用户需要显式点击“加载模型”后才能开始翻译。

## 核心用户流程

### 1. 导入模型

1. 用户点击“导入模型”。
2. 平台弹出系统文件选择器。
3. 用户选择 `.gguf` 模型文件。
4. App 将模型复制到平台私有模型目录。
5. App 校验文件类型和 GGUF magic header。
6. 成功后模型面板显示模型文件名。
7. 用户点击“加载模型”。

平台目录要求：

- Android：`files/models/`
- iOS / macOS：`Application Support/ai_offline_translator/models/`

### 2. 下载模型

1. 用户点击“下载模型”。
2. App 从内置默认模型源下载 `Hy-MT1.5-1.8B-STQ1_0.gguf`。
3. 下载过程中展示进度。
4. 下载过程中允许用户取消。
5. 取消后删除临时文件，保持原模型选择不变。
6. 下载完成后校验 `.gguf` 扩展名、GGUF magic header 和最小文件大小。
7. 校验通过后保存到平台私有模型目录。
8. 模型面板显示模型文件名。
9. 用户点击“加载模型”。

默认模型：

```text
Hy-MT1.5-1.8B-STQ1_0.gguf
```

默认下载源：

```text
https://modelscope.cn/models/AngelSlim/Hy-MT1.5-1.8B-1.25bit-GGUF/resolve/master/Hy-MT1.5-1.8B-STQ1_0.gguf
```

### 3. 加载与翻译

1. 用户在模型面板确认当前模型名称。
2. 用户点击“加载模型”。
3. 加载成功后，用户输入原文。
4. App 按 200 有效字符限制输入；空格和符号不计入限制。
5. 用户点击“翻译”。
6. App 展示译文。
7. 翻译完成后，用户可以复制译文。

## 跨端一致性要求

- Android、iOS、macOS 的产品行为必须一致。
- 各端都不能提供手填路径输入框。
- 各端都不能在普通界面展示完整本地路径。
- 各端都必须把模型落到 App 私有目录。
- 各端都必须支持导入模型。
- 各端最终都应支持下载默认模型、下载进度、取消下载和基础文件校验。
- 各端加载模型前必须有明确的用户动作，不做静默自动加载。

## 错误与状态文案

### 未选择模型

```text
请先导入或下载 GGUF 模型。
```

### 模型未加载

```text
请先加载模型。
```

### 当前模型为空

```text
未选择模型
```

### 模型选择说明

未选择时：

```text
请导入本地 GGUF，或从 ModelScope 下载默认模型
```

已选择时：

```text
模型已保存到 App 私有目录
```

## 非目标

- 不做通用模型路径输入。
- 不做任意 URL 下载输入。
- 不在首版做多模型市场。
- 不把 GGUF 模型提交到 Git。
- 不把 CLI `llama-completion` 作为移动端产品架构。

## 技术约束

- 推理路线必须服务移动端落地：Flutter UI + shared native `translator_engine` + thin platform bridge。
- 当前默认模型只支持已验证的 `Hy-MT1.5-1.8B-STQ1_0.gguf`。
- `third_party/llama.cpp` 必须保持支持 `STQ1_0` 的 PR #22836 兼容路径。
- 原生引擎必须复用 `llama.cpp/common` 的 chat template、tokenize、sampler 和 token-to-piece 能力。
- Android 首个 native 目标只支持 `arm64-v8a`。
